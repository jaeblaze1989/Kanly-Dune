RELEASE NOTES

Kanly Changelog

Highlights of changes shipped in the latest update cycle.

v1.2.0

2026-05-23
  • Added dashboard quick action to check for Git updates directly from the control panel.
  • Added authenticated API endpoint /api/system/update/check with branch and commit comparison details.
  • Made update checks read-only-safe by comparing local commits against origin using git ls-remote.
  • Container runtime now includes Git and mounts the Kanly repo for in-app update visibility.
  • Fixed server command execution environment by installing Bash in the runtime image.

v1.1.0

2026-05-23
  • Dashboard and auth pages now use full-width page shells for a wider workspace layout.
  • Removed Director API polling and the Director API tab from the dashboard to keep monitoring database-first.
  • setup-docker.sh install now prompts for the dune-awakening-selfhost-docker path when KANLY_DUNE_ROOT is not set.
  • README now recommends Install From GitHub as the primary installation flow.
  • Added setup-docker.sh update to self-update from Git (fast-forward), rebuild, and restart Kanly.

v1.0.0

Initial release
  • Go-based Kanly Manager with auth setup/login flow.
  • Dashboard modules for host stats, containers, map INI editing, database insights, and command execution.
